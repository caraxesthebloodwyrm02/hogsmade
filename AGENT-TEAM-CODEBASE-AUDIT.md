# Agent Team Codebase Audit & Showcase Prompt
 
## Mission Briefing for Claude Code Agent Team
 
You are leading an elite agent team to conduct a comprehensive audit and showcase of the CascadeProjects codebase. This mission requires parallel exploration, collaborative synthesis, and the creation of compelling demonstrations that highlight the ecosystem's capabilities and future potential.
 
### Team Structure & Roles
 
**Lead Agent**: You orchestrate the overall audit, coordinate teammates, and synthesize findings into actionable insights.
 
**Specialized Teammates**:
1. **Architect Analyst** - Deep dive into code architecture, patterns, and design decisions
2. **Capability Demonstrator** - Create working examples that showcase each project's power
3. **Use Case Architect** - Design realistic scenarios and step-by-step guides
4. **Future Vision Strategist** - Analyze trends and craft forward-looking recommendations
5. **Integration Specialist** - Map connections and create cross-project demonstrations
 
### Phase 1: Comprehensive Codebase Audit (Parallel Exploration)
 
**Create agent team with 5 teammates for parallel codebase analysis:**
 
```
I need you to create an agent team to audit the CascadeProjects codebase comprehensively. 
 
Spawn 5 specialized teammates:
1. Architect Analyst - examine code structure, patterns, and technical debt
2. Capability Demonstrator - identify and document core capabilities 
3. Use Case Architect - map user journeys and application scenarios
4. Future Vision Strategist - analyze trends and evolution opportunities
5. Integration Specialist - map interconnections and cross-project synergies
 
Each teammate should work independently on their domain, then share findings for collaborative synthesis. Focus on GRID-main and glimpse-engine as primary projects, with MCP servers and supporting projects as secondary.
 
Start with parallel exploration, then converge on discussion and insights.
```
 
**Key Audit Areas**:
- **Architecture Quality**: Design patterns, modularity, scalability
- **Code Health**: Test coverage, complexity metrics, technical debt
- **Capability Maturity**: Feature completeness, production readiness
- **Integration Points**: Cross-project dependencies and synergies
- **Future Readiness**: Alignment with emerging trends and technologies
 
### Phase 2: Collaborative Synthesis & Discussion
 
**After initial exploration, facilitate team discussion:**
 
```
Team, now that you've each explored your domains, let's synthesize findings.
 
Architect Analyst: Share your assessment of the codebase's structural strengths and weaknesses. What architectural patterns stand out? Where are the technical debt hotspots?
 
Capability Demonstrator: What are the most impressive capabilities you discovered? Which features would blow users away?
 
Use Case Architect: What user journeys did you identify? Who benefits most from these tools?
 
Future Vision Strategist: How does this ecosystem align with AI/development trends? Where are the biggest opportunities?
 
Integration Specialist: How do these projects amplify each other? What cross-project capabilities would be game-changing?
 
Debate and challenge each other's findings. Build a comprehensive picture of where we stand and where we could go.
```
 
**Discussion Focus Areas**:
- **Strengths & Weaknesses**: Honest assessment of current state
- **Competitive Advantages**: What makes this ecosystem unique
- **Critical Gaps**: Missing capabilities or architectural issues
- **Synergy Opportunities**: How projects amplify each other
- **Market Positioning**: Where this fits in the broader tool landscape

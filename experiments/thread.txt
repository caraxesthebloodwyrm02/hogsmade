What This Is
A trigger system that decides when, why, and how a notification fires. Not a UI for displaying notifications — the logic layer underneath.

Core Components
1. Trigger Definition

A trigger is a condition-action binding. Every trigger has:

Field	Type	Description	Example
id	string	Unique trigger ID	trg_upwork_connects_low
source	enum	Where the signal originates	calendar, system, threshold, pattern, manual
condition	object	Evaluation rule	{ metric: "connects_remaining", op: "<=", value: 3 }
action	object	What fires when condition is true	{ type: "alert", channel: "push", priority: "high" }
cooldown_ms	int	Min time between re-fires	86400000 (24h)
context	object	Metadata passed to the notification renderer	{ sprint_week: 2, phase: "P1" }

**3. Evaluation Engine**
```
┌─────────────┐     ┌──────────────┐     ┌─────────────┐     ┌──────────────┐
│ Signal       │ ──→ │ Condition    │ ──→ │ Cooldown     │ ──→ │ Action       │
│ Ingestion    │     │ Evaluator    │     │ Gate         │     │ Dispatcher   │
└─────────────┘     └──────────────┘     └─────────────┘     └──────────────┘
       │                   │                    │                    │
  event_bus           rule_match?          last_fired +          channel routing
  poll/subscribe      true → pass          cooldown < now?       push/log/ui/escalate
```

**Evaluation order matters:**
1. Signal arrives (event bus message, cron tick, manual invoke)
2. All registered triggers for that source type are evaluated
3. Conditions matched → cooldown check
4. Cooldown passed → action dispatched
5. `last_fired` timestamp updated

**4. Priority Tiers**

| Priority | Behavior | Example |
|---|---|---|
| `critical` | Interrupt immediately, no suppression | Confidence score collapse |
| `high` | Push + badge, suppresses lower priorities in same window | Sprint at risk, gate failure |
| `medium` | Standard push, respects quiet hours | Upcoming outreach window |
| `info` | Log only, visible on review | Git merge, step completion |
| `silent` | Write to event store, no notification | Telemetry, heartbeat |

**5. Conflict Resolution**

When multiple triggers fire simultaneously:
- **Same channel, same priority:** bundle into single notification with item list
- **Same channel, different priority:** highest priority wins, others queued
- **Competing actions (alert vs. suppress):** explicit suppress wins (user-defined quiet hours override everything except `critical`)
